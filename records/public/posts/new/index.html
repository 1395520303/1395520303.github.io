使用Prisma连接mysql

2022年11月20日·2 分钟 草稿

第一次写博客有点不知所措，随便聊聊吧😋
之前跟着学Nestjs的时候视频里用了prisma，当时听的似懂非懂，感觉Nestjs好用又不好用，Nestjs没记住多少但是记住了prisma，所以现在想仔细学一下
本篇文章只是介绍一下prisma连接数据库（我用的是mysql）以及一些简单的操作（复杂的我暂时还用不上），所以如果有讲的不好的请多多见谅🤧

注意 本教程需要安装Node.js（12.2及以上）

1.安装Prisma #

先建立一个文件夹,这里我就建立一个叫做prisma-mysql的文件夹

mkdir prisma-mysql 
cd prisma-mysql

然后我们初始化一下并添加依赖

npm init -y
npm i prisma -D

现在我们就可以调用Prisma CLI了，通过前缀npx来调用Prisma CLI:

npx prisma

通过使用以下命令创建Prisma schema文件来设置Prisma项目

npx prisma init

这个Prisma schema是什么呢
咱们使用 npx prisma init 会建立一个名为prisma的新目录，其中包含一个名为schema.prisma的文件和一个位于项目根目录中的.env文件
schema.prisma中包含了prisma模式，简单来说就是连接什么数据库，还有prisma客户端生成器，这个我觉得不用了解，知道就好😁
.env是一个用于定义环境变量的文件（用于数据库连接）。

2.连接数据库 #

prisma/schema.prisma #

datasource db {
    provider = "postgresql"
    url      = env("DATABASE_URL")
}

嗯，这就是你应该在schema.prisma中应该看到的东西🧐，因为prisma默认使用的是postgresql，所以刚刚你用的init命令就建立了postgresql的连接，现在我们把它改成mysql的连接

datasource db {
    provider = "mysql"
    url      = env("DATABASE_URL")
}

怎么样，是不是还挺简单的

下面我们设置一下这个DATABASE_URL，如果你已经看过了.env文件，那你应该对这个东西不是很陌生。要连接mysql数据库，这个url的结构应该是这个样子的

mysql://Username:Password@Host:Port/Database

本地运行时，你的URL通常是这个样子的

DATABASE_URL="mysql://root:randompassword@localhost:3306/mydb"

3.创建表单 #

接下来我们建一些表玩一下这个prisma，就用官方给出的例子吧

prisma/schema.prisma #

model Post {
  id        Int      @id @default(autoincrement())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  title     String   @db.VarChar(255)
  content   String?
  published Boolean  @default(false)
  author    User     @relation(fields: [authorId], references: [id])
  authorId  Int
}

model Profile {
  id     Int     @id @default(autoincrement())
  bio    String?
  user   User    @relation(fields: [userId], references: [id])
  userId Int     @unique
}

model User {
  id      Int      @id @default(autoincrement())
  email   String   @unique
  name    String?
  posts   Post[]
  profile Profile?
}

这些数据模型需要映射到数据库中，我们使用prisma migrateCLI命令

npx prisma migrate dev --name init

这个name你可以自己取，它用于给你这次的记录命名
现在我们的表单也创建好了，下面我们就要往表里添加数据并进行操作了！😋

4.添加数据并进行操作！ #

要开始使用Prisma Client，你需要安装@prisma/client包

npm install @prisma/client

安装命令会自动调用prisma generate，它会读取prisma模式并生成一个适合您的模型的prisma Client版本。

这个generate命令嘛，官方的解释我还不是很懂，不过用我的理解来讲（比较浅显），就是它会根据你在schema.prisma文件里的模型去数据库里生成对应的表，有时间我再去看看吧🤣

下面我们在根目录中建立一个index.js文件

index.js #

const { PrismaClient } = require('@prisma/client')

const prisma = new PrismaClient()

async function main() {
    const allUsers = await prisma.user.findMany()
    console.log(allUsers)
}

main()
  .catch((e) => {
    throw e
  })
  .finally(async () => {
    await prisma.$disconnect()
  })

接着我们执行下面的命令

node index.js

结果应该是一个空的数组，很明显，我们没有添加数据嘛

我们在原有的基础上加上一些代码

await prisma.user.create({
    data: {
        name: 'Alice',
        email: 'alice@prisma.io',
        posts: {
            create: { title: 'Hello World' },
        },
        profile: {
            create: { bio: 'I like turtles' },
        },
    },
})

这就是添加数据的操作

完整版： #

async function main() {
  await prisma.user.create({
    data: {
      name: 'Alice',
      email: 'alice@prisma.io',
      posts: {
        create: { title: 'Hello World' },
      },
      profile: {
        create: { bio: 'I like turtles' },
      },
    },
  })

  const allUsers = await prisma.user.findMany({
    include: {
      posts: true,
      profile: true,
    },
  })
  console.dir(allUsers, { depth: null })
}

我们看到的结果应该是这样的：

[
  {
    email: 'alice@prisma.io',
    id: 1,
    name: 'Alice',
    posts: [
      {
        content: null,
        createdAt: 2020-03-21T16:45:01.246Z,
        id: 1,
        published: false,
        title: 'Hello World',
        authorId: 1,
      }
    ],
    profile: {
      bio: 'I like turtles',
      id: 1,
      userId: 1,
    }
  }
]

查看数据库应该也能看到我们添加了相对应的数据

下面再看看修改的操作

async function main() {
  const post = await prisma.post.update({
    where: { id: 1 },
    data: { published: true },
  })
  console.log(post)
}

你将看到以下输出

{
    id: 1,
    title: 'Hello World',
    content: null,
    published: true,
    authorId: 1
}

怎么样，是不是感觉prisma还是挺有意思的😊

5.在浏览器中查看数据库 #

当时让我印象最深刻的还是prisma的这个功能，它可以在浏览器中看数据库耶！

你只要在终端中输入以下命令

npx prisma studio

就可以在浏览器中查看你的数据库，可以在表中进行增删改查的操作，很方便不是吗？
